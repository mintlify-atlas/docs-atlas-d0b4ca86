The VSpeak frontend is a React application built with Vite, TypeScript, and Tailwind CSS. It provides the IDE interface with Monaco Editor, terminal emulator, and walkthrough player.

## Local Development

<Steps>
  <Step title="Install Dependencies">
    Navigate to the web directory and install packages:
    
    ```bash
    cd web
    npm install
    ```
    
    **Key dependencies:**
    - `react` & `react-dom` - UI framework
    - `@monaco-editor/react` - Code editor
    - `@xterm/xterm` - Terminal emulator
    - `vite` - Build tool and dev server
    - `typescript` - Type safety
    - `tailwindcss` - Styling
  </Step>

  <Step title="Configure API Endpoint">
    The development server proxies API requests to the backend:
    
    ```typescript vite.config.ts
    export default defineConfig({
      plugins: [react()],
      server: {
        port: 3000,
        proxy: {
          '/api': {
            target: 'http://localhost:8000',
            changeOrigin: true,
          },
        },
      },
    })
    ```
  </Step>

  <Step title="Start Development Server">
    ```bash
    npm run dev
    ```
    
    The app will be available at `http://localhost:3000`
  </Step>
</Steps>

## Production Build

<Steps>
  <Step title="Build for Production">
    ```bash
    npm run build
    ```
    
    This command:
    1. Runs TypeScript compiler (`tsc -b`)
    2. Builds optimized bundle with Vite
    3. Outputs to `dist/` directory
    
    <Note>
    The build process includes:
    - Code minification
    - Tree shaking
    - Asset optimization
    - Source map generation
    </Note>
  </Step>

  <Step title="Preview Build Locally">
    Test the production build:
    
    ```bash
    npm run preview
    ```
    
    This serves the `dist/` directory at `http://localhost:4173`
  </Step>

  <Step title="Configure API URL">
    For production, update the API URL. Create a `.env.production` file:
    
    ```bash
    VITE_API_URL=https://api.vspeak.example.com
    ```
    
    Or set the backend URL in your code:
    
    ```typescript
    const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:8000';
    ```
  </Step>
</Steps>

## Deployment Options

### Option 1: Static File Hosting

Deploy the `dist/` folder to any static hosting service.

<CodeGroup>
```bash Netlify
# Install Netlify CLI
npm install -g netlify-cli

# Deploy
netlify deploy --prod --dir=dist
```

```bash Vercel
# Install Vercel CLI
npm install -g vercel

# Deploy
vercel --prod
```

```bash AWS S3
# Build
npm run build

# Upload to S3
aws s3 sync dist/ s3://your-bucket-name --delete

# Set bucket for static website hosting
aws s3 website s3://your-bucket-name \
  --index-document index.html \
  --error-document index.html
```

```bash Google Cloud Storage
# Build
npm run build

# Upload to bucket
gsutil -m rsync -r -d dist/ gs://your-bucket-name

# Set bucket for static website
gsutil web set -m index.html -e index.html gs://your-bucket-name
```
</CodeGroup>

### Option 2: Nginx Static Server

<Steps>
  <Step title="Build Application">
    ```bash
    npm run build
    ```
  </Step>

  <Step title="Copy to Web Root">
    ```bash
    sudo cp -r dist/* /var/www/vspeak/
    ```
  </Step>

  <Step title="Configure Nginx">
    Create `/etc/nginx/sites-available/vspeak`:
    
    ```nginx
    server {
        listen 80;
        server_name vspeak.example.com;
        
        # Redirect to HTTPS
        return 301 https://$server_name$request_uri;
    }

    server {
        listen 443 ssl http2;
        server_name vspeak.example.com;
        
        ssl_certificate /etc/ssl/certs/vspeak.crt;
        ssl_certificate_key /etc/ssl/private/vspeak.key;
        
        root /var/www/vspeak;
        index index.html;
        
        # Enable gzip compression
        gzip on;
        gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
        gzip_vary on;
        
        # Cache static assets
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
        
        # SPA routing - serve index.html for all routes
        location / {
            try_files $uri $uri/ /index.html;
        }
        
        # Proxy API requests to backend
        location /api {
            proxy_pass http://127.0.0.1:8000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
        
        # WebSocket support
        location /ws/ {
            proxy_pass http://127.0.0.1:8000;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }
    ```
  </Step>

  <Step title="Enable Site">
    ```bash
    sudo ln -s /etc/nginx/sites-available/vspeak /etc/nginx/sites-enabled/
    sudo nginx -t
    sudo systemctl reload nginx
    ```
  </Step>
</Steps>

### Option 3: Docker with Nginx

<Steps>
  <Step title="Create Dockerfile">
    Create `web/Dockerfile`:
    
    ```dockerfile
    # Build stage
    FROM node:18-alpine AS build

    WORKDIR /app

    # Copy package files
    COPY package*.json ./
    RUN npm ci

    # Copy source code
    COPY . .

    # Build application
    RUN npm run build

    # Production stage
    FROM nginx:alpine

    # Copy built assets from build stage
    COPY --from=build /app/dist /usr/share/nginx/html

    # Copy custom nginx config
    COPY nginx.conf /etc/nginx/conf.d/default.conf

    EXPOSE 80

    CMD ["nginx", "-g", "daemon off;"]
    ```
  </Step>

  <Step title="Create Nginx Config">
    Create `web/nginx.conf`:
    
    ```nginx
    server {
        listen 80;
        server_name localhost;
        
        root /usr/share/nginx/html;
        index index.html;
        
        # Gzip compression
        gzip on;
        gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
        
        # Cache static assets
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
        
        # SPA routing
        location / {
            try_files $uri $uri/ /index.html;
        }
    }
    ```
  </Step>

  <Step title="Build and Run">
    ```bash
    # Build image
    docker build -t vspeak-frontend .

    # Run container
    docker run -d \
      --name vspeak-frontend \
      -p 3000:80 \
      vspeak-frontend
    ```
  </Step>
</Steps>

### Option 4: Full Stack Docker Compose

Deploy both frontend and backend together:

```yaml docker-compose.yml
version: '3.8'

services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    env_file:
      - ./backend/.env
    volumes:
      - ./workspace:/app/workspace
      - ./sessions:/app/sessions
    restart: unless-stopped

  frontend:
    build:
      context: ./web
      dockerfile: Dockerfile
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    ports:
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./ssl:/etc/ssl:ro
    depends_on:
      - frontend
      - backend
    restart: unless-stopped
```

Start everything:
```bash
docker-compose up -d
```

## Build Optimization

<Accordion title="Reduce Bundle Size">
  Configure Vite for optimal bundle size:
  
  ```typescript vite.config.ts
  import { defineConfig } from 'vite'
  import react from '@vitejs/plugin-react'
  
  export default defineConfig({
    plugins: [react()],
    build: {
      rollupOptions: {
        output: {
          manualChunks: {
            'monaco': ['@monaco-editor/react', 'monaco-editor'],
            'xterm': ['@xterm/xterm', '@xterm/addon-fit'],
            'react-vendor': ['react', 'react-dom'],
          },
        },
      },
      chunkSizeWarningLimit: 1000,
    },
  })
  ```
</Accordion>

<Accordion title="Enable Code Splitting">
  Use React lazy loading:
  
  ```typescript
  import { lazy, Suspense } from 'react'
  
  const Editor = lazy(() => import('./components/Editor'))
  const Terminal = lazy(() => import('./components/Terminal'))
  const WalkthroughPlayer = lazy(() => import('./components/WalkthroughPlayer'))
  
  function App() {
    return (
      <Suspense fallback={<Loading />}>
        <Editor />
        <Terminal />
        <WalkthroughPlayer />
      </Suspense>
    )
  }
  ```
</Accordion>

<Accordion title="Optimize Assets">
  Configure asset handling:
  
  ```typescript vite.config.ts
  export default defineConfig({
    build: {
      assetsInlineLimit: 4096, // Inline assets < 4kb
      cssCodeSplit: true,
      sourcemap: false, // Disable in production
    },
  })
  ```
</Accordion>

## SSL/HTTPS Setup

<Steps>
  <Step title="Obtain SSL Certificate">
    Use Let's Encrypt with Certbot:
    
    ```bash
    sudo apt-get install certbot python3-certbot-nginx
    sudo certbot --nginx -d vspeak.example.com
    ```
  </Step>

  <Step title="Auto-renewal">
    Certbot automatically sets up renewal. Test it:
    
    ```bash
    sudo certbot renew --dry-run
    ```
  </Step>

  <Step title="Configure Nginx for SSL">
    Certbot updates Nginx config automatically, or manually add:
    
    ```nginx
    ssl_certificate /etc/letsencrypt/live/vspeak.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/vspeak.example.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build fails">
    - Clear node_modules and reinstall: `rm -rf node_modules package-lock.json && npm install`
    - Check TypeScript errors: `npm run lint`
    - Ensure Node.js version is 18+
  </Accordion>

  <Accordion title="API requests fail">
    - Verify VITE_API_URL is set correctly
    - Check CORS settings in backend
    - Ensure backend is accessible from frontend server
    - Check browser console for errors
  </Accordion>

  <Accordion title="Monaco Editor not loading">
    - Ensure Monaco assets are included in build
    - Check network tab for 404s on Monaco files
    - Verify base URL is configured correctly
  </Accordion>

  <Accordion title="WebSocket connection fails">
    - Check if HTTPS is enforced (use wss:// for WebSocket)
    - Verify Nginx WebSocket upgrade headers
    - Check browser console for connection errors
  </Accordion>
</AccordionGroup>
