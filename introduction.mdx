# Welcome to VSpeak

**Watch code write itself while an AI voice explains every step in perfect sync.**

VSpeak transforms any coding task into a **cinematic, voice-narrated tutorial**. Think Cursor IDE meets YouTube tutorials — the AI writes code, creates files, runs commands, and explains everything in real-time with perfect audio-visual synchronization.

## Architecture Overview

VSpeak uses a **pluggable pipeline architecture** where each component is independent and swappable:

| Component | What It Does | Swap In Any... |
|-----------|--------------|----------------|
| **LLM** | Generates script + IDE actions | OpenAI, Anthropic, Gemini, Llama, Mistral, etc. |
| **TTS** | Synthesizes expressive voice | ElevenLabs, OpenAI TTS, Azure, Coqui, Bark, etc. |
| **STT** | Extracts word timestamps for sync | Whisper (any variant), Deepgram, AssemblyAI, etc. |

## How It Works

<Steps>
  <Step title="Generate Script">
    The LLM creates a narrative script with timestamped IDE actions (create files, type code, run commands).
  </Step>
  <Step title="Synthesize Audio">
    TTS converts the script into expressive voice narration. Long scripts are automatically chunked and stitched seamlessly.
  </Step>
  <Step title="Extract Timestamps">
    Whisper STT processes the audio to extract word-level timestamps for perfect synchronization.
  </Step>
  <Step title="Sync Actions">
    The timeline engine maps IDE actions to audio timestamps, creating a single synchronized playback.
  </Step>
</Steps>

## Key Features

### Audio-Driven Timeline

Unlike screen recordings, VSpeak uses the **audio element as the master clock**:
- All IDE actions fire relative to `audio.currentTime`
- Seeking instantly reconstructs IDE state at any point
- No drift between audio and visual elements

### Interactive Chapters

Navigate your walkthrough with automatically generated chapters:
- Click any chapter to jump instantly
- IDE state reconstructs from scratch at the new position
- Perfect for reviewing specific sections

### Full IDE Integration

- **Monaco Editor** — The same editor powering VS Code
- **Xterm.js Terminal** — Real terminal with command execution
- **File Explorer** — Browse and manage your workspace
- **Chat Panel** — Ask questions and generate new walkthroughs
