The VSpeak API is built with FastAPI and provides endpoints for chat assistance, walkthrough generation, file management, and terminal execution.

## Base URL

All API endpoints are served from:

```
http://localhost:8000
```

## Authentication

Currently, the API does not require authentication. All endpoints are publicly accessible when running locally.

<Warning>
  This API is designed for local development. Do not expose it to the public internet without adding proper authentication and security measures.
</Warning>

## Response Format

All API responses follow standard HTTP status codes:

- **200 OK** - Successful request
- **404 Not Found** - Resource not found
- **400 Bad Request** - Invalid request parameters
- **409 Conflict** - Resource already exists
- **500 Internal Server Error** - Server error

## Error Responses

Errors return a JSON object with a `detail` field:

```json
{
  "detail": "Error message describing what went wrong"
}
```

## CORS

The API allows all origins (`*`) for local development:

```python
allow_origins=["*"]
allow_credentials=True
allow_methods=["*"]
allow_headers=["*"]
```

## WebSocket Endpoints

VSpeak also provides WebSocket endpoints for real-time communication:

- **`ws://localhost:8000/ws/terminal`** - Interactive terminal sessions

See the [Terminal API](/api/terminal) documentation for WebSocket details.

## Data Models

All request and response models are defined using Pydantic and can be found in `backend/models/types.py`. Key models include:

- **ChatMessage** - Individual chat message
- **ChatSession** - Chat conversation with history
- **WalkthroughSession** - Complete walkthrough with audio and actions
- **SyncedAction** - IDE action synchronized to audio timestamp
- **Chapter** - Timeline marker for walkthroughs

## Health Check

<api method="GET" endpoint="/api/health">
  Check the API server status and get basic system information.
</api>

### Response

```json
{
  "status": "healthy",
  "workspace": "/path/to/workspace",
  "sessions_count": 3,
  "walkthroughs_count": 5
}
```

<ResponseField name="status" type="string">
  Always `"healthy"` if the server is responding.
</ResponseField>

<ResponseField name="workspace" type="string">
  Absolute path to the configured workspace directory.
</ResponseField>

<ResponseField name="sessions_count" type="number">
  Number of active chat sessions in memory.
</ResponseField>

<ResponseField name="walkthroughs_count" type="number">
  Number of walkthroughs currently loaded in memory.
</ResponseField>

### cURL Example

```bash
curl http://localhost:8000/api/health
```

## Rate Limiting

Currently, there are no rate limits. Commands have a 30-second timeout for safety.
