Chapters divide walkthroughs into logical sections, making it easy to jump to specific topics, review key concepts, and understand the overall structure at a glance.

## Overview

Chapters provide:
- **Visual timeline**: See the entire walkthrough structure
- **Quick navigation**: Jump to any section instantly
- **Progress tracking**: Know where you are in the tutorial
- **Hover previews**: See chapter details before seeking

## Chapter Data Structure

Chapters are defined during script generation:

```python
# From backend/models/types.py
class Chapter(BaseModel):
    id: str
    title: str  # "Setup the Project"
    description: Optional[str]  # "Install dependencies and create files"
    start_chunk_index: int  # Which script chunk starts this chapter
    end_chunk_index: int  # Which script chunk ends this chapter
    start_ms: Optional[int]  # Calculated after audio generation
    end_ms: Optional[int]  # Calculated after audio generation
```

### Timestamp Calculation

Chapter timestamps are calculated based on chunk positions:

```python
# From backend/server.py
for chapter in chapters_with_snapshots:
    start_word_idx = int(chapter.start_chunk_index * words_per_chunk)
    end_word_idx = int((chapter.end_chunk_index + 1) * words_per_chunk) - 1
    
    if start_word_idx < len(result.timestamps):
        chapter.start_ms = result.timestamps[start_word_idx].start_ms
    if end_word_idx < len(result.timestamps):
        chapter.end_ms = result.timestamps[end_word_idx].end_ms
```

## Seeking Behavior

When you click a chapter:

<Steps>
  <Step title="Seek to timestamp">
    Audio player jumps to `chapter.start_ms`
  </Step>
  
  <Step title="Rebuild IDE state">
    Action executor replays all actions up to that point in batch mode
  </Step>
  
  <Step title="Resume playback">
    If playing, continue from the new position; if paused, stay paused
  </Step>
</Steps>

```typescript
// From web/src/editor/actionExecutor.ts
beginBatch() {
  this.isBatch = true;
  this.cancelTyping();  // Apply actions instantly during seek
}

endBatch() {
  this.isBatch = false;
}
```

<Note>
Batch mode ensures seeking is fast and deterministic—no animations, just instant state reconstruction.
</Note>

## Chapter Snapshots

Chapters can include IDE state snapshots for instant restoration:

```python
# From backend/models/types.py
class IDEStateSnapshot(BaseModel):
    files: dict[str, str]  # filename -> content
    open_tabs: list[str]
    active_file: Optional[str]
    cursor_position: Optional[dict[str, int]]  # {line, column}
    terminal_output: list[str]

class ChapterWithSnapshot(Chapter):
    snapshot: Optional[IDEStateSnapshot] = None
```

<Tip>
Snapshots make seeking to later chapters faster by avoiding full action replay.
</Tip>

## Time Formatting

Chapter times are displayed in `M:SS` format:

```typescript
// From web/src/components/ChapterTimeline.tsx
function formatTime(ms: number): string {
  if (!Number.isFinite(ms) || ms < 0) return '0:00';
  const totalSeconds = Math.floor(ms / 1000);
  const minutes = Math.floor(totalSeconds / 60);
  const seconds = totalSeconds % 60;
  return `${minutes}:${seconds.toString().padStart(2, '0')}`;
}
```

Examples:
- `0:00` - Start
- `1:23` - 1 minute, 23 seconds
- `12:05` - 12 minutes, 5 seconds

### Scrubbing the Timeline

```typescript
const handleMouseMove = (e: React.MouseEvent) => {
  const rect = timelineRef.current.getBoundingClientRect();
  const x = e.clientX - rect.left;
  const percentage = Math.max(0, Math.min(1, x / rect.width));
  const hoverTimeMs = percentage * totalDurationMs;
  
  // Find chapter at this time
  const chapter = chapters.find(ch =>
    hoverTimeMs >= ch.start_ms && hoverTimeMs < ch.end_ms
  );
  
  setHoveredChapter(chapter ?? null);
  
  if (isDragging) {
    onSeek(hoverTimeMs);  // Live scrubbing
  }
};
```
